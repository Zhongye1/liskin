5. 你可以怎么理解“依次创建”的架构顺序

按能力依赖，真实顺序是：

1. 协议入口（router/handler）
2. 治理前置（decorate: auth/access/quota）
3. 转发编排（logic/proxy）
4. provider 适配（openrouter/modelhub→ark）
5. 计费上下文化（billing_decision + gwcontext）
6. 观测与企业化增强（fornax/metrics/internal_source/async）
