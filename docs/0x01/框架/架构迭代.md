可以看出的“搭建节奏”

- 第一阶段（2025  
  Q2）：先把基础服务骨架立起来（agent/prd/agw/codebase/codesearch）。
- 第二阶段（2025  
  Q2-Q3）：补业务能力和资源/计费链路（coding/d2c/asset/expense/faas）。
- 第三阶段（2025  
  Q3-Q4）：异步任务、文档检索、运维能力完善（async/docsearch/ops）。
- 第四阶段（2026 Q1-Q2）：围绕 LLM 与会话体系集中建设（contextengine +
  modelgateway + session_gateway + botassistant）。

当前仓落地说明（2026-06）：

- 由于 `server/application/model_gateway` 暂无可运行实现，统一入口先在 `server/application/faasrouter` 落地。
- 新增 `POST /gateway/model` 作为单路径入口，兼容 OpenAI + Anthropic。
- 协议识别规则：`x-protocol` / `x-api-format` 显式优先，body 结构兜底。
- `POST /forward`、`POST /forward/stream` 作为迁移期兼容入口保留。
