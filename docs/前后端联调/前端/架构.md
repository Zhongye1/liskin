对话框实现

对话框的核心是一条时间线——用户输入和 assistant 输出（文本 + 工具调用）按到达顺序交织排列。

数据模型

// events.ts — 纯 reducer，EventMsg 流 → Turn 树

Turn {                              // 一轮对话 = 一次 user 发送
  id: string       4567890
  userContent: string                // "修复 src/utils.ts"
  steps: Step[]    text + tool 交织
  status: 'running' | 'done' | 'interrupted' | 'error'
}

Step =                              // 时间线上的一个块
  | TextStep { kind: 'text', id, parts: string[] }  // assistant
文本流
  | ToolStep {                      // 工具调用
      kind: 'tool', id, call: ToolCall,
      status: 'pend| 'done' | 'error',
      stdout: string[], stderr: string[],
      result?: ToolResult
    }

事件流 → Turn 的 re

EventMsg 流
──────────────────────────────────────────────────────────
TurnStart  ─────────────────→  turn.status = 'running'
Token("我来看看…") "我来看看…")
ToolCall(fs.read)  ──────────→  push tool step (status: pending)
ToolProgress("line 1\n")  ───→  tool.stdout.push, status → running
ToolResult(ok, "...., status → done
Token("问题在第 3 行")  ───────→  push 新 text step（与 tool 交织）
TurnEnd(completed)  ─────────→  turn.status = 'done'
