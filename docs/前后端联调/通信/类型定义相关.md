KernelClient 接口的版本差异怎么处理？

当前 web 的 HttpSseKernelClient 需要 getRecord(sessionId) 方法和简化的 submit({ sessionId, content }) 签名——这跟 protocol 的 KernelClient 不完全一致。有两个选择：

1. 把 getRecord 加进 protocol 的 KernelClient——如果 server 端的 InProcessKernelClient 也需要这个方法
2. Web 定义自己的 WebKernelClient extends KernelClient——只加 web 需要的方法，不污染共享接口

推荐选项 1，因为 getRecord 本质上是 StorePort.loadSession 的对外暴露，server 端也应该有。
