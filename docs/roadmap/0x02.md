根据项目文档和代码库的全面分析，以下是 Liskin 目前尚未完成的内容，按优先级排列：

---

Phase 0 已完成（M0–M4）✅

内核状态机、OpenAI Provider、工具系统/沙箱、Hono daemon、CLI 入口、Web 前端（原型版）均已完成，134 个测试全过。

---

Phase 1 — 高优先级未完成项

1. 🔴 Web 前端重写（三步走，文档完备但未开工）

当前 Web 前端是 Phase 0 遗留版本，存在三个核心问题：

┌──────────┬──────────────────────────────────────────────────────────────────┬─────────────────────────────────────────────────────────────────────────────────────────┐
│ 问题 │ 现状 │ 目标 │
├──────────┼──────────────────────────────────────────────────────────────────┼─────────────────────────────────────────────────────────────────────────────────────────┤
│ 协议错位 │ 直连旧 POST /v1/chat，未用 KernelClient 接口 │ 升级为 POST /v1/sessions/:id/turns SSE，web 并入 workspace，从 @liskin/core import 类型 │
├──────────┼───────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────┤
│ 渲染割裂 │ 文本流和工具调用分属不同 UI 区域 │ 统一时间线渲染（markdown/代码高亮/diff/终端式输出） │
├──────────┼───────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────┤
│ 状态双轨 │ 散装 useState + 假消息 hack（确认用 <continue:id> 伪 user 消息） │ HttpSseKernelClient + Zustand store，Turn/Step reducer │
└──────────┴──────────────────────────────────────────────────────────────────┴─────────────────────────────────────────────────────────────────────────────────────────┘

详见 docs/architecture/web-frontend-design.md。

2. 🔴 CLI 交互式 REPL（agent chat）

当前 CLI 只有 agent exec（一次性 headless 任务），缺少交互式对话模式。文档规划优先于 Web 重写，因为 CLI 能 in-process 验证 KernelClient 协议，无需 HTTP/SSE 链路。

3. 🟡 MCP 协议支持

ToolPort 接口已预留，MCP 客户端只需作为另一个 ToolPort 实现注入。需支持 stdio 和 HTTP 两种 transport。这是 Phase 1 价值最大的增量项。

4. 🟡 多 Provider 扩展

当前只有 OpenAI 兼容适配器。LLMPort 接口天然支持新增 Anthropic、Google 等 Provider。

5. 🟡 Harness 框架

将当前 NoopHarness 占位替换为 MarkdownHarness：

- 在 .liskin/harness/active/ 下创建 Markdown 任务文档
- 7 个工具：harness_create、record_node_result、update_graph_plan 等
- 支持任务中断续跑、审计、恢复

6. 🟡 项目记忆（AGENTS.md）

让 agent 启动时读取并遵守项目根目录的 AGENTS.md 约定文件。

7. 🟡 沙箱执行加固

- --sandbox 标志接入 OS 级隔离（Linux bubblewrap / macOS sandbox-exec）
- 写/删文件前 diff 预览
- 撤销栈

8. 🟢 Agent Loop 完善

Read → Plan → Act → Verify 完整循环，工具集扩展（list_files、grep、edit 等）。

9. 🟢 协议升级 v2

docs/architecture/kernel-protocol.md 中设计的 ContentBlock union 协议、三段式 tool_use 事件、ToolSource 抽象层。

10. 🟢 收尾清理

清理未使用依赖（ahooks、axios、部分 @radix-ui 等），bundle 拆分（react-markdo 除 614KB chunk）。

---

Phase 2 — 条件触发（后端需求出现时才做）

- 可观测性：结构化日志 (JSONL session transcripts)、OpenTelemetry Tracing、Token 用量计量
- 多 Agent 协作：子 Agent 定义 (TOML)、并行执行 + 结果聚合、MCP Server 模式
- 从 @liskin/core 拆出独立的 @liskin/protocol 包

---

Phase 3+ — 可能永远不做

- 后端网关：Key 托管、多用户、审计、限流（激活搁置的 Go server/ 代码）
- IDE 插件：VSCode / JetBrains 插件，连接 agent serve daemon
- 工作流编排：多 Agent DAG 执行

---

建议执行顺序

文档中推荐的 Phase 1 顺序是：

▎ CLI REPL → 协议对齐 → Web 重写 → 工具集扩展 → MCP → 多 Provider → Harness → 沙箱加固

核心逻辑是 先 CLI 验证协议，再 Web 跟进，避免 Web 重写踩坑。MCP 放在工具集扩展之后是因为内置工具先跑通，再对外开放。
